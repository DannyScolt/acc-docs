## ADDED Requirements

### Requirement: Phase 2 SHALL provide a standardized overview section and module matrix
The standalone Phase 2 document SHALL begin with an approved-style overview including a process diagram and module matrix so readers can understand scope, sequencing, outputs, and milestones before reading detailed modules.

#### Scenario: Overview is visible before detail
- **WHEN** a reader opens the standalone Phase 2 document
- **THEN** the document SHALL present an overview process schematic and a module matrix before the detailed module specifications

### Requirement: Phase 2 SHALL reuse the common document-state control model
The standalone Phase 2 document SHALL include a shared document-state section and state legend modeled after Phase 1, adapted to the set of procurement, inventory, invoice, and payment documents used in Phase 2.

#### Scenario: Shared control flow is documented
- **WHEN** a reader reviews the common control-flow section of the Phase 2 document
- **THEN** the document SHALL describe the baseline states, allowed transitions, and key controls in a legend table

### Requirement: Phase 2 SHALL normalize appendices for rules, accounting entries, handover results, and acceptance criteria
The standalone Phase 2 document SHALL end with structured appendix material covering business rules, sample accounting entries, handover outcomes, and acceptance criteria in a format comparable to Phase 1.

#### Scenario: Appendices are implementation-ready for review
- **WHEN** a reader reaches the end of the standalone Phase 2 document
- **THEN** the document SHALL provide structured business rules, accounting-entry guidance, handover outputs, and acceptance checks suitable for accounting review and downstream implementation preparation
